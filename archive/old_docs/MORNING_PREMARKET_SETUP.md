# 아침 프리마켓 스캐너 사용법

## 개요

**전날 밤** 단기 종목 선정 → **당일 아침 8시** 프리마켓 데이터로 재점검 → **8시 50분** 최종 후보 텔레그램 전송

---

## 실행 방법

### 1. 전날 밤 (또는 아침 7시) - 기술적 후보 선정

```bash
python run_short_term.py --top 30 --score 50
```

- **결과**: `symposium/screener/data/short_term/2026-06-25.json`에 30개 후보 저장

---

### 2. 당일 아침 - 프리마켓 스캔

#### 옵션 A: 자동 스케줄 실행 (추천)

```bash
python run_morning_premarket.py --schedule
```

**동작**:
- 8:00, 8:10, 8:20, 8:30, 8:40, 8:50에 자동 스캔
- 8:50 최종 스캔 후 텔레그램 전송
- 자동 종료

#### 옵션 B: 즉시 1회 실행

```bash
python run_morning_premarket.py --now
```

#### 옵션 C: 시간 감지 모드 (기본)

```bash
python run_morning_premarket.py
```

- 8:00~8:59: 즉시 실행
- 그 외: 8:00까지 대기 후 스케줄 시작

## 점수 체계 (프리마켓 추가)

### 기존 점수 (전날 밤 선정)
- 진입 타이밍: 35점
- 테마: 25점
- 수급: 15점
- 실적: 15점
- 밸류에이션: 10점

### 프리마켓 점수 (당일 아침 추가)
- 갭상승 +5% 이상: **+15점**
- 갭상승 +3~5%: **+12점**
- 갭상승 +1~3%: **+8점**
- 매수 호가 우위: **+5점**
- 갭하락 -3% 이상: **-5점**

**최종 점수 = 기존 점수 + 프리마켓 점수**

## 자동 실행 설정 (선택)

### Windows 작업 스케줄러

1. 작업 스케줄러 열기
2. "기본 작업 만들기"
3. 트리거: 매일 07:55
4. 작업: `python C:\autotrade\run_morning_premarket.py --schedule`

### Linux/Mac cron

```bash
crontab -e
```

```
55 7 * * 1-5 cd /path/to/autotrade && python run_morning_premarket.py --schedule
```

## 데이터 소스

### 네이버 금융 (무료, API 없음)
- **예상 체결가**: 8:30부터 확인 가능
- **동시호가**: 8:30~9:00
- **한계**: 실시간 호가창은 JS 동적 로딩

- **한국투자증권 API**: REST API, 프리마켓 데이터

## 문제 해결

### "전날 후보 종목 없음"
→ `python run_short_term.py` 먼저 실행

### "프리마켓 시간 외"
→ 8:00~8:59 사이에만 실행됨

### "텔레그램 전송 실패"
→ `.env`에 `TELEGRAM_TOKEN`, `TELEGRAM_CHAT_ID` 확인
